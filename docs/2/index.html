This section will cover how to set up the debugger for this workbook as well as future workbooks and other projects you may want to debug in the future.

I also recommend only having a single folder in your workspace being this one, this may make the process below easier.

Setting up

For this setup, we’re going to be using Chrome but this also works with many other browsers.

Run the live server for your page (It should already be running for this Debugger WB).

Click on the run/debug tab.

The next step may look a little different depending on whether or not you already have debug launch configs set up for debugging.

If there are no launch configs

Your debug tab should look like the image below. Click on Run and Debug.

If there are already launch configs

The debug tab will look like the image below, containing other launch options. Go to the top and click the scrolldown. Click on Add Config ([Name of Workbook]).

For this workbook, you would click Add Config (DebuggerTutorial21).

The steps after should be the same

It will then ask you to select an environment, in which you will just select what browser you use to open your page, which for this example will be Chrome (It might be Chrome (legacy)). If you didn’t install the Debugger for Chrome, Chrome will not show up as an option.

Now this launch.json file should open up and it can be found in the newly added .vscode folder of this workbook. We need to modify the url so that the port matches the one we are using for the live server. You can also change the name of the config if you wish, but in this example, we will not do that.

We are now done with setting up the debugger for VS Code.

If instead, it never asked you to select an environment and took you straight to the launch.json asking you to select a config like in the image below,

that means you already have a config for the folder and a launch.json file. You should delete the launch.json file the follow the steps above starting from after getting the Debugger for Chrome. There are other ways to handle this, but for those who are new to this, this would be the simplest way to handle this.

Note: we only have to do this once per project folder. If you open up a project that you already set up the debugger for, the launch settings will have been saved in launch.json in the .vscode folder in. This effects the entire folder, you don’t have to worry about doing this for each individual file.

Running the debugger

The new launch config should now show up when you click the dropdown. It should show as:

Launch Chrome against localhost (DebuggerTutorial21).

Select this and try running the debugger. It can only run when the live server is open. Make sure you have the correct debugger config selected as shown before. You can run the debugger by clicking the arrow next to your launch tab.

It should open a new window with your page site. In VS Code, you should see this debugger bar at the top of VS Code. You’ll also notice that the bar at the bottom turns orange.

When do breakpoints activate?

Breakpoints pause the program when the line of the breakpoint is reached. So if you have already reached the end of the program and then you add a breakpoint, it won’t do anything since you already went past the breakpoint, you would have to refresh the page to reach it again.

A breakpoint can still activate if you add it after starting your program without having to refresh. This would occur in things like mouse events or animation loops where functions may get called again after time has passed, and if you placed a breakpoint in those repeated functions before it gets called, it will pause when it does. We can see this later in section 4’s exercise.

Why are my breakpoints greyed out?

Below is an image of what breakpoints look like when they are greyed out.

This simply means that the code area is not being loaded, (I.E. the page containing the Code is not the currently visited page). They will turn red when you go to the page containing the code with the breakpoints.

This could also mean you are debugging in the wrong launch config or have some kind of issue with the config. If you are having issues with the config, the simplest thing to do would be to the delete the .vscode folder in your project and redo the steps above on this page to create the config.
